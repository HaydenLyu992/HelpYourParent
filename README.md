# HelpYourParent · 康护亲

> 基于 HealthKit 健康数据的老年人远程健康守护 App

子女不在身边，父母的健康谁来守护？**康护亲** 通过 HealthKit 实时读取老人的心率、血氧、心电图、跌倒检测、睡眠、运动等健康数据，结合其身高、体重、既往病史、当前服用药物的个人健康档案，通过 DeepSeek V4 Flash 大模型进行智能风险分析。一旦发现疑似异常（如心律不齐、血氧骤降、跌倒、长期不动），系统立即通过 **短信 / 邮件** 通知绑定的健康守护者（子女或紧急联系人），让关爱跨越距离。

---

## 核心功能

### 健康数据采集（通过 HealthKit）
- 实时心率、静息心率、心率变异性（HRV）
- 血氧饱和度（SpO₂）
- 心电图（ECG）
- 跌倒检测与 SOS
- 睡眠分析与睡眠呼吸频率
- 步数、活动能量、站立时长
- 环境音量（听力健康）

### 个人健康档案
- 基本信息：年龄、身高、体重、血型
- 既往病史：高血压、糖尿病、心脏病、脑卒中等
- 当前用药：药物名称、剂量、服用频率
- 过敏史与手术史

### 智能风险分析
- 结合健康档案基线，对实时健康数据做多维度分析
- 内置算法识别以下风险场景：
  - 心率异常（过高 / 过低 / 心律不齐）
  - 血氧过低（< 95% 进入关注，< 90% 触发告警）
  - 跌倒或疑似跌倒
  - 长期无活动（超出阈值时间无运动数据）
  - 睡眠呼吸异常
- 规则引擎 + Core ML 本地推理 + DeepSeek 语义理解

### 分级告警
- 🟢 **绿色**：一切正常，每日健康摘要
- 🟡 **黄色**：轻度异常，App 内提示 & 推送通知
- 🟠 **橙色**：中度风险，短信通知守护者
- 🔴 **红色**：高风险（如疑似跌倒、严重心律异常），短信 + 邮件 + 电话级提醒

### 用户注册登录
- 手机号 + 阿里云短信验证码登录，免密码，老年友好
- 老人与守护者两种角色，支持扫码绑定
- 一位老人可绑定多位守护者，一位守护者可看护多位老人

### AI 健康助手（DeepSeek V4 Flash）
- **风险解读**：健康数据异常时，AI 自动生成通俗易懂的风险解释和就医建议
- **用药咨询**：基于当前用药列表，AI 提醒药物相互作用和服用注意事项
- **健康报告**：每日/每周 AI 生成自然语言健康摘要，守护者一眼看懂
- **智能问诊**：老人可用语音或文字描述不适，AI 结合健康档案给出个性化建议

### 通知偏好设置
- 每位守护者可独立配置通知渠道：短信 / 邮件 / App 推送，可单独开关
- 三种渠道均可关闭，关闭后仅在 App 内展示静默通知
- 老人本人也可设置自己的通知偏好（如仅在 App 内查看每日健康摘要）

### 守护者系统
- 每位老人可绑定多位守护者（子女、亲属、邻居、社区医生）
- 守护者接收分级告警通知
- 守护者可随时查看老人近期健康摘要（非全部原始数据，隐私保护）

### 技术栈

**前端 (iOS)**

| 层级 | 技术选型 |
|------|---------|
| UI | SwiftUI + Combine |
| 健康数据 | HealthKit (`HKHealthStore`) |
| 本地存储 | SwiftData |
| 本地 AI | Core ML |
| 网络 | URLSession + Codable |

**后端**

| 层级 | 技术选型 |
|------|---------|
| 语言 & 框架 | Java 17 + Spring Boot 3.x |
| 微服务 | Spring Cloud Alibaba (Nacos + Sentinel + Gateway) |
| 鉴权 | Spring Security + JWT |
| 数据库 | 阿里云 RDS MySQL 8.0 |
| 缓存 | 阿里云 Redis 7.0 |
| ORM | Spring Data JPA + QueryDSL |
| 短信 | 阿里云短信 SDK (通过 FC 异步调用) |
| 邮件 | 阿里云邮件推送 SDK (通过 FC 异步调用) |
| AI 代理 | 阿里云函数计算 (DeepSeek 网关) |
| 部署 | 阿里云 ECS (单体初期) / ACK (微服务期) |

```

┌────────────┐        ┌──────────┐ ┌──────────┐
│            │        │ 用户服务  │ │ 告警服务  │
```

## 隐私与安全

- 所有原始健康数据仅存储在用户设备本地
- 风险分析在设备本地完成，原始数据不上传服务器
- 守护者仅能看到"风险摘要"和"健康趋势概要"，无法访问原始数据
- 短信/邮件告警仅包含风险类型和建议措施，不传输具体健康指标
- DeepSeek 查询数据经脱敏处理，不含可识别个人信息
- 后端 AK/SK 统一走 Nacos 配置中心或函数计算环境变量，代码中无硬编码
- 符合 Apple HealthKit 隐私政策要求
